# הקמת ה-Backend (Google Sheets + Apps Script)

מדריך חד־פעמי. אחרי זה, כל הוספה של מושג/שאלה נשמרת אוטומטית בגיליון ומחכה לאישור.

## שלב 1 — יצירת הגיליון
1. היכנסו ל־[sheets.new](https://sheets.new) (נפתח גיליון חדש).
2. תנו לו שם, למשל: **Industry 4.0 — מאגר מושגים**.

## שלב 2 — הדבקת הקוד
1. בגיליון: תפריט **Extensions → Apps Script**.
2. מחקו את כל הקוד שמופיע (`function myFunction() {}`).
3. העתיקו את כל התוכן של [`Code.gs`](Code.gs) והדביקו במקום.
4. לחצו **Save** (אייקון הדיסקט).

## שלב 3 — יצירת הגיליונות
1. בעורך ה-Apps Script, בחרו בתפריט הפונקציות את `setup` ולחצו **Run**.
2. בפעם הראשונה Google יבקש הרשאות: **Review permissions → בחרו את החשבון → Advanced → Go to (project name) → Allow**.
3. אמורה להופיע הודעה שהגיליונות מוכנים. בגיליון ייווצרו שתי לשוניות: `terms` ו-`questions`.

## שלב 4 — הגדרת קוד מנהל (לאישור שאלות)
1. בעורך Apps Script: **Project Settings** (גלגל שיניים בצד) → גללו ל-**Script Properties** → **Add script property**.
2. **Property:** `ADMIN_KEY`
3. **Value:** בחרו סיסמה כלשהי (למשל `sagiv2026`) — זה הקוד שהמרצה/אתה תזינו באפליקציה כדי לאשר.
4. **Save script properties**.

## שלב 5 — פרסום כ-Web App
1. לחצו **Deploy → New deployment**.
2. ליד "Select type" לחצו על גלגל השיניים ⚙ → בחרו **Web app**.
3. הגדירו:
   - **Description:** `industry40 api`
   - **Execute as:** `Me`
   - **Who has access:** `Anyone`  ← חשוב! בלי זה האתר לא יוכל לקרוא.
4. **Deploy** → אשרו הרשאות אם נדרש.
5. **העתיקו את ה-Web app URL** (נראה כמו `https://script.google.com/macros/s/AKfy.../exec`).

## שלב 6 — חיבור האפליקציה
1. פתחו את האפליקציה → טאב **"ניהול והוספה"** → אזור **"חיבור לענן"**.
2. הדביקו את ה-URL מהשלב הקודם, ואת קוד המנהל (אם אתם המאשרים).
3. לחצו **שמור וסנכרן**. אמור להופיע "סונכרן בהצלחה".

## מצב אישור למרצה (קישור ייעודי)

יש מסך אישור נקי שמראה **רק** את הפריטים הממתינים עם כפתורי ✔ אשר / ✖ דחה:

**https://bohanyzahal-cyber.github.io/industry40-glossary/?approve**

שולחים למרצה את הקישור הזה + את קוד המנהל (בהודעה פרטית). הוא מזין את הקוד פעם אחת ורואה את התור.

> ⚠️ הפעולה **"דחה"** והצגת **"הוצע ע״י"** דורשות את גרסת ה-`Code.gs` המעודכנת. אם עדכנתם את הקוד — בצעו פרסום מחדש (ראו "עדכון הקוד" בתחתית). האישור עצמו עובד גם בלי.

## איך המרצה מאשר

**אפשרות א' — בגיליון (הכי פשוט עבורו):**
פותחים את לשונית `questions`, ובעמודת **status** משנים את הערך מ-`ממתין` ל-**`מאושר`**. זהו. השאלה תיכנס לתרגול אצל כולם בסנכרון הבא.

**אפשרות ב' — באפליקציה:**
בטאב "ניהול" מזינים את קוד המנהל (`ADMIN_KEY`), ואז כפתור "אשר" ליד כל שאלה ממתינה עובד ישירות מול הגיליון.

## הערות אבטחה
- **הוספה** פתוחה לכל מי שיש לו את הקישור — אבל כל הוספה נכנסת כ**"ממתין"** ולא משפיעה על אף אחד עד אישור.
- **אישור** מוגן ב-`ADMIN_KEY` — בלי הקוד אי אפשר לאשר כלום. אל תשתפו את הקוד בקבוצת הוואטסאפ.

## עדכון הקוד בעתיד
אם משנים את `Code.gs`, צריך לפרסם מחדש: **Deploy → Manage deployments → ✏️ → Version: New version → Deploy**. ה-URL נשאר זהה.
